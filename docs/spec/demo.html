Public API

The python server allows for a clean public api, while restricting public access to the raw database. This api can be called via ajax.

  • http://socialfarm.org/api/businesses
  • http://socialfarm.org/api/business/testbus
  • http://socialfarm.org/api/business/testbus/members
  • http://socialfarm.org/api/business/testbus/actions
  • http://socialfarm.org/api/business/testbus/jobs
  • http://socialfarm.org/api/business/testbus/tasks

Still in Progress...

PUT requests are functional but I have none available to show today. OSN specific content has yet to be added to the database. The goal is to only return OSN specific content when necessary, i.e. views for inviting facebook or twitter friends to join a business, or other network specific activities.

Building and Testing Related

We are currently using a makefile which calls serveral python scripts which access couchdb via its python api. Project files are uploaded from the db directory.

There are also some test scripts used to check the status of RESTful calls to couchdb and the python server.
